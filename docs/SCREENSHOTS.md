# LexAI Screenshot Guide

Screenshots should be stored in:

```text
assets/screenshots/
```

Use demo data or a test account only. Do not show real secrets, private contracts, API keys, tokens, or `backend/.env`.

| Signup | `signup.png` |

## Capture Tips

- Capture at a 1440px desktop width for portfolio consistency.
- Keep browser zoom at 100%.
- Use the seeded demo workspace or a test account.
- Avoid showing real secrets, customer data, or local machine credentials.
- Do not include `backend/.env` in screenshots.
- Prefer clean states where the main product value is visible.
- Use consistent naming so README image links stay stable.
- If replacing screenshots, keep old file names when possible to avoid broken GitHub image references.

## Suggested README Set

For a concise GitHub presentation, start with these four:

- `dashboard-auth.png`
- `analysis.png`
- `ai-chat.png`
- `report-detail.png`

Then add the rest as the portfolio page or repository matures.
